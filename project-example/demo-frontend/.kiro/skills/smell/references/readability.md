# Readability Checklist

Evaluate how easy the code is to read and follow:

- **Naming** — Are variables, functions, and classes named clearly and consistently?
- **Flow** — Can a reader follow the logic without jumping around?
- **Abstraction level** — Does each function operate at a single level of abstraction?
- **Formatting** — Is indentation, spacing, and structure consistent?
- **Cognitive load** — How much context does a reader need to hold in memory?
- **Self-documenting** — Does the code explain itself without excessive comments?
