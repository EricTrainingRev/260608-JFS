# Issues

## Creating Issues

- Use clear, descriptive titles that summarize the problem or feature request.
- Include enough context in the body for someone unfamiliar to understand the issue.
- Apply labels when appropriate (bug, enhancement, documentation, etc.).
- Assign to a team member if the user specifies, otherwise leave unassigned.

## Searching & Filtering Issues

- Search by label, assignee, milestone, or state (open/closed).
- When the user asks about "open bugs" or "my issues," translate that into the appropriate filter.

## Managing Issues

- **Closing**: Close issues when the related work is merged or the issue is resolved.
- **Labeling**: Add/remove labels as the issue evolves.
- **Commenting**: Add comments to provide updates, ask for clarification, or link related PRs.
- **Milestones**: Associate issues with milestones when the user is tracking releases or sprints.

## Best Practices

- Don't create duplicate issues — search first to see if one already exists.
- Reference related issues or PRs in comments to maintain traceability.
- Keep issue threads focused; open a new issue for tangential discussions.
